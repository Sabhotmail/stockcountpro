Security MVP

Date: 2026-07-15 Status: Approved Scope: Count integrity + authz polish (no offline Dexie)

Goals / Non-goals

Goals

  1. clientMutationId idempotency — กัน retry ซ้ำ save / audit
  2. Block submit จนกว่า autosave จะครบ (client flush + readiness)
  3. HTTP status: deny → 403, not found → 404
  4. Approve + FinalCountEntry ใน transaction เดียว

Non-goals

  • Offline IndexedDB / Dexie
  • Express change-warning entity
  • Full audit forensic fields (ip / userAgent)
  • Role consolidation

Current state

  • Zod รับ clientMutationId แล้ว แต่ service ไม่ใช้
  • Summary ส่งได้แม้หน้านับยังมี save ค้าง
  • หลาย route map error → 400 เสมอ
  • snapshotFinalCountEntries อยู่นอก transaction เดียวกับ COMPLETED

User workflow (ที่ต้องไม่พัง)

  1. พนักงานกรอกจำนวน → autosave (retry ได้โดยไม่ซ้ำ audit)
  2. กดไปหน้าสรุป → ระบบรอ flush queue ก่อน
  3. ยืนยันส่งหัวหน้า → ได้เฉพาะ DRAFT ที่บันทึกครบ
  4. หัวหน้าอนุมัติ → COMPLETED + final snapshot พร้อมกัน
