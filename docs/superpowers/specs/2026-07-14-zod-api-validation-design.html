Zod API Request Validation

Date: 2026-07-14 Owner: StockCount Pro Scope: Validate JSON request bodies at Next.js Route Handlers with Zod

Goals / Non-goals

Goals

  • ติดตั้ง zod เป็น dependency โดยตรงของโปรเจกต์
  • parse + validate body ที่ขอบ API ด้วย safeParse แทน as / เช็ก typeof มือ
  • คงรูป error เดิม: { "error": string } + HTTP 400 เมื่อ shape ไม่ถูกต้อง
  • business rules (permission, quantity, lock) ยังอยู่ใน service เหมือนเดิม

Non-goals

  • ไม่ validate response ของ Express upstream ด้วย Zod ในรอบนี้
  • ไม่เปลี่ยน UI / ไม่เปลี่ยน message ที่ service สร้างเอง
  • ไม่บังคับ client-side Zod forms ในรอบนี้

Current state

  • zod ยังไม่ได้อยู่ใน package.json — resolve จากแอปไม่ได้
  • Route ที่รับ body ใช้ (await request.json()) as … หรือเช็กมือ (เช่น branches, sync locations)
  • มี ~15 handlers ที่อ่าน JSON body: auth, express sync, entries, note, recount, approve, push bulk, admin CRUD

User workflow (developer / request path)

  1. Client ส่ง JSON ไปยัง Route Handler
  2. Handler อ่าน raw body (ดัก invalid JSON → 400)
  3. รัน Zod schema ที่ตรงกับ endpoint
  4. ถ้า fail → 400 { error } ไม่เรียก service
  5. ถ้าผ่าน → ส่ง typed data เข้า service เหมือนเดิม
request.json() → schema.safeParse → service / 400

UI Details

ไม่มีหน้า UI ใหม่ — แสดงพฤติกรรมที่ client เห็นเมื่อ body ไม่ผ่าน validation (toast / network error จาก API เดิม)

Invalid body — API 400 · ทุก role ที่เรียก endpoint

POST /api/express/sync · Staff / Counter / Manager / Admin / HQ
Request
{ "date": "2026-07-14", "locations": "WH01" }
Response 400
{ "error": "locations: Expected array, received string" }

หน้า Sync แสดง error จาก response เหมือนกรณี validation อื่น — ไม่มี dialog ใหม่

Login missing fields — /login · Guest

POST /api/auth/login
400 {"error":"Username and password are required"}

คงข้อความภาษาอังกฤษเดิมที่ login ใช้อยู่แล้ว เพื่อไม่เปลี่ยน UI copy ของหน้า login

Approve optional body — Supervisor

POST /api/supervisor/count-documents/[id]/approve
OK empty / non-JSON body → pushToExpress = false (พฤติกรรมเดิม)

เฉพาะ endpoint นี้ที่ยอม empty body; endpoint อื่น invalid JSON = 400

  • src/lib/api/schemas.ts — schemas ต่อ endpoint (login, sync, entry, batch, note, recount, approve, push-bulk, admin*)

Error handling & test checklist

  • Invalid JSON → 400 Invalid JSON body (ยกเว้น approve)
  • Schema fail → 400 พร้อม message จาก field แรก (สั้น อ่านง่าย)
  • Unit tests แบบ node assert สำหรับ helper + schemas หลัก (sync locations, login, entry)
  • npm run build ผ่านหลังเปลี่ยน route handlers
  • Manual smoke: login, sync preview POST, save entry, create branch
