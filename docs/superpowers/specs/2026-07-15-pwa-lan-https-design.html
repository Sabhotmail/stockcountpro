PWA + LAN HTTPS — Design Spec

Date: 2026-07-15 Status: Pending approval Scope: Installable tablet app + UI shell cache (not offline counting)

1. Goal

  • Tablet ติดตั้ง StockCount Pro บน Home Screen ได้ (standalone)
  • เปิดแอปเร็วขึ้น / มีหน้า shell เมื่อเน็ตหลุดชั่วคราว
  • ใช้งานหลักบน LAN ด้วย HTTPS self-signed (mkcert) + Caddy

Out of scope

  • Offline นับสต็อก / คิว PATCH entries (Dexie) — ไม่ทำในรอบนี้
  • Push notifications
  • บังคับเลิกใช้ npm start แบบ HTTP เดิม (เก็บไว้สำหรับ dev/debug)

2. Architecture — HTTPS on LAN

Tablet → https://<LAN-IP>:3443 → Caddy (mkcert TLS)
                                   ↓ reverse_proxy
                                next start 127.0.0.1:3000
  • mkcert: สร้าง cert สำหรับ IP ของเครื่อง (และ localhost) — ติดตั้ง root CA บน tablet ครั้งเดียว
  • Caddyfile ใน repo (เช่น deploy/Caddyfile): listen :3443, proxy ไป 127.0.0.1:3000, ส่งต่อ X-Forwarded-Proto
  • สคริปต์/เอกสาร: npm run start:https หรือคู่มือสั้นรัน Next + Caddy คู่กัน
  • Session cookie: secure context จาก HTTPS — สอดคล้อง logic shouldUseSecureCookies + x-forwarded-proto

3. PWA surface

3.1 Web app manifest

  • ใช้ App Router src/app/manifest.ts (Next Metadata Route)
  • name / short_name: StockCount Pro
  • start_url: /login (หลังติดตั้้งเปิดที่ login)
  • display: standalone
  • lang: th, theme_color / background_color โทนแอปปัจจุบัน
  • ไอคอน: เพิ่ม public/icons/icon-192.png และ icon-512.png (สร้างจากแบรนด์ง่าย ๆ / placeholder ที่ชัดเจน)

3.2 Metadata / viewport

  • อัปเดต src/app/layout.tsx: manifest, appleWebApp, themeColor
  • คง viewportFit: cover เดิมสำหรับ safe-area บน tablet

3.3 Service worker (Serwist)

  • ใช้ Serwist ที่รองรับ Next 16 / Turbopack (@serwist/turbopack + worker source)
  • Route handler สร้าง SW (เช่น /serwist/sw.js)
  • SerwistProvider ใน layout (client bridge)
  • Middleware/proxy.ts: ไม่บังคับ session บน path ของ SW / static icons

3.4 Caching policy (ระดับ 2)

ResourceStrategyหมายเหตุ
Precache app shell assets (build-hashed JS/CSS, icons, offline page) precache อัปเดตตาม build revision
Navigations ที่ล้มเหลวเพราะ offline fallback → /~offline หน้าไทยสั้น ๆ บอกให้ต่อเน็ต — ไม่พยายามนับต่อ
/api/* Network only ห้าม cache auth/count mutations
เอกสาร count / entries ไม่ cache เป็น offline writable store ชัดเจนว่าอยู่นอกสcope

4. UX

  • หลังติดตั้ง: ไอคอนเปิดเข้า /login แบบ standalone (ไม่มี browser chrome)
  • ถ้า offline แล้วเปิด route ที่ต้องโหลดเครือข่าย → แสดงหน้า /~offline
  • ไม่ใส่ install banner/custom promo ที่รก UI — พึ่ง native browser install

5. Ops / docs

  • docs/pwa-lan-https.md (หรือส่วนใน README): ติดตั้ง mkcert, trust CA บน Android tablet, รัน Caddy, URL ที่ใช้
  • .gitignore: ไม่ commit private keys ของ mkcert ถ้าเก็บใต้โฟลเดอร์ local (prefer ใช้ mkcert ของเครื่อง)
  • ตัวอย่าง Caddyfile ใช้ placeholder IP / ตัวแปรที่แก้ได้

6. Testing

  1. npm run build && npm start + Caddy ด้วย cert mkcert
  2. เปิด https://<LAN-IP>:3443 บน tablet (CA trusted) → Install app
  3. ยืนยัน SW registered ใน DevTools (desktop Chrome ผ่าน HTTPS)
  4. Airplane mode → เปิดแอปแล้วเห็น /~offline ไม่พังขาว
  5. ออนไลน์ → login + นับจำนวนทำงานเหมือนเดิม (PATCH entries ยังผ่าน network)
  6. HTTP npm start เดิมยังใช้ได้ (SW อาจไม่ register — ไม่ error)

7. Success criteria

  • ติดตั้ง PWA ได้บน tablet ที่ trust mkcert CA
  • มี offline shell page; ไม่เคลมว่า offline counting พร้อมใช้
  • API count/auth ไม่ถูก cache ผิด
  • เอกสารรัน HTTPS LAN ชัดเจน
