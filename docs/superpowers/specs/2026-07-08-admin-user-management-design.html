Admin User Management (Design Spec)

Date: 2026-07-08 Owner: StockCount Pro Scope: Admin UI + API จัดการผู้ใช้ (สร้าง/แก้ไข/ปิดใช้งาน/รีเซ็ตรหัสผ่าน)

Goals

  • หน้า Admin จัดการผู้ใช้: สร้าง, แก้ไข (ชื่อ, role, สาขา), ปิด/เปิดใช้งาน, รีเซ็ตรหัสผ่าน
  • รหัสผ่าน: ตั้งเอง หรือ สุ่ม (แสดงครั้งเดียวให้ copy)
  • ผู้ใช้ที่ปิดใช้งาน login ไม่ได้ (พฤติกรรมเดิม)

Non-goals

  • เปลี่ยนรหัสผ่านด้วยตนเอง (self-service)
  • ส่งรหัสผ่านทาง Email/SMS
  • Audit log การแก้ user (phase 2)

API Design

MethodEndpointPurpose
GET/api/admin/usersรายการผู้ใช้
POST/api/admin/usersสร้างผู้ใช้
PATCH/api/admin/users/[userId]แก้ไข + isActive
POST/api/admin/users/[userId]/reset-passwordรีเซ็ตรหัสผ่าน

POST create body

{
  "name": "string",
  "username": "string",
  "role": "STAFF",
  "branchIds": ["branch_bkk1"],
  "passwordMode": "set" | "generate",
  "password": "optional when set"
}

Validation Rules

  • username — trim, lowercase, unique
  • password — ความยาวขั้นต่ำ (เช่น 8+) · hash ด้วย hashPassword
  • branchIds — บังคับอย่างน้อย 1 สาขาสำหรับ role ที่ไม่ใช่ ADMIN/HQ
  • ห้ามปิดใช้งานตัวเอง (session.userId === targetUserId)

Test Plan

  1. สร้าง user ตั้งรหัสเอง → login ได้
  2. สร้าง user สุ่มรหัส → copy แล้ว login ได้
  3. แก้ role/สาขา → สิทธิ์เปลี่ยนตาม
  4. ปิดใช้งาน → login ไม่ได้ · เปิดใหม่ → login ได้
  5. รีเซ็ตรหัสผ่าน → รหัสเก่าใช้ไม่ได้
